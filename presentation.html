👤 Introduction & Background

Background

// About This Project
  • Deepfakes advancing faster than biometric defenses
  • Fusion systems assume both modalities are genuine
  • Goal: quantify exactly how systems fail when one modality is forged
  • Bridges deepfake detection + biometric verification
  • One cohesive pipeline: attack characterization → failure quantification → defense
  • Face + voice fakes achievable with minimal data
  • Deployed systems in banking, healthcare, enterprise auth
  • Partial deepfake (1 modality) often bypasses fusion systems

⚠️ Problem Statement

The Verification Assumption is Breaking Down

// The Implicit Assumption
  • AV systems trust that face & voice belong to same genuine person
  • Fusion score = weighted sum of both modality scores
  • No mechanism to detect partial manipulation
// Real-World Deployments
  • 🏦 Remote eKYC & bank account opening
  • ☎️ Call-center customer authentication
  • 🔐 Enterprise secure conferencing
  • 💳 Step-up auth for high-risk transactions
// The Attack Gap
  • Voice clone possible from seconds of audio
  • Face-swap achievable with consumer tools
  • Single-modality attack → fused score still above threshold
  • System incorrectly ACCEPTS the attacker
// Core Research Question
  • How do AV fusion systems fail under modality-specific deepfake substitution?
  • How do we detect and defend against it?

🏗️ System Architecture

5-Module Pipeline

M1
ENROLL
Enrollment Engine
  • Registers face + voice templates from short enrollment clip
  • MagFace (face) · ECAPA-TDNN (voice) — generate embeddings stored in secure database
M2
VERIFY
Unimodal Verification Branches
  • Face: RetinaFace → align → MagFace embed → cosine sim + quality q_face
  • Voice: Silero VAD → ECAPA-TDNN embed → cosine sim + quality q_voice
M3
FUSE
Fusion Module — 3 Strategies
  • Score Fusion — fixed α-weighted sum
  • Learned MLP — on [s_face, s_voice, q_face, q_voice]
  • RJCA — recursive joint cross-attention (research baseline)
M4
ATTACK
Deepfake Attack Test Harness
  • 5 controlled scenarios S0–S4 · FakeAVCeleb + ASVspoof + FF++ + WaveFake
  • Metrics: FAR · ASR · EER · ECE
M5
POLICY
Deepfake-Aware Decision Policy
  • ACCEPT — fused score high & deepfake prob low
  • REJECT — deepfake prob high (even if score is high)
  • STEP-UP — quality low or score ambiguous

📐 Evaluation Metrics

How We Measure Success

// Biometric Verification Metrics
  • FAR — False Accept Rate: fraction of impostors incorrectly accepted
  • EER — Equal Error Rate: threshold where FAR = FRR (lower is better)
  • FRR — False Reject Rate: fraction of genuine users incorrectly rejected
  • ECE — Expected Calibration Error: measures how well confidence scores match actual accuracy
// What We Are Optimising For
  • Minimise FAR ↑ under deepfake attack (don't let fakes in)
  • Maintain low FRR for genuine users (don't lock real users out)
  • Keep ECE low — system confidence should be calibrated and trustworthy

📄 Related Work — Paper 1

Audio-Visual Fusion via Cross-Attention

IEEE FG 2024
Audio-Visual Person Verification based on Recursive Fusion of Joint Cross-Attention (RJCA)
// Key Idea
  • Face embeddings iteratively query voice segments and vice versa
  • Each recursive pass uncovers complementary cross-modal cues
  • Verification via cosine-similarity on fused representation
  • Consistent gains over score-fusion and MLP baselines on VoxCeleb
✅ Strengths
  • Fine-grained inter-modal correspondences
  • Modular: any encoder plugs in
  • Strong benchmark results
❌ Limitations
  • No adversarial or deepfake evaluation
  • High compute: limits real-time use
🎯 Our use: RJCA = cross-attention fusion baseline — stress-tested against all S0–S4 deepfake scenarios

📄 Related Work — Paper 2

Multimodal Deepfake Dataset

NeurIPS 2021 Datasets & Benchmarks
FakeAVCeleb: A Novel Audio-Video Multimodal Deepfake Dataset
// Key Contributions
  • First dataset with all 4 AV manipulation combos: real/fake audio × real/fake video
  • Video fakes: face-swap + re-enactment · Audio fakes: voice conversion
  • Seeded from VoxCeleb celebrities
  • Shows single-modality detectors fail when both are manipulated
✅ Strengths
  • First to cover all 4 AV manipulation combos
  • Systematic S0–S3 scenario coverage
  • VoxCeleb-aligned
❌ Limitations
  • Request-only access
🎯 Our use: Test dataset for all attack scenarios: video-only, audio-only, both-fake, and cross-identity substitution

Audio Spoofing & Deepfake Speech Detection

IEEE/ACM TASLP 2022
ASVspoof 2021: Accelerating Progress in Spoofed and Deepfake Speech Detection
// Key Contributions
