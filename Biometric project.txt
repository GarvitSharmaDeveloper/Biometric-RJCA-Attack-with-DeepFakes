Project Documentation
1) Project title
Robust Audio-Visual Biometric Verification Under Modality-Specific Deepfakes using Uncertainty-Aware Fusion and Deepfake-Aware Decision Policies

2) Why this topic matters
Audio-visual biometrics (face + voice) is increasingly relevant for remote onboarding, secure video calls, customer support verification, and device re-auth. In practice, these systems face two big realities:
Modalities fail asymmetrically (bad lighting vs noisy audio).
Deepfakes can selectively target one modality (voice clone with real face, or face swap with real voice).
Most fusion systems still behave as if “both signals are trustworthy,” which creates a clear research gap: fusion can become overconfident and accept attackers when one stream is manipulated.

3) Where it is used
eKYC / remote onboarding (banking, telecom, fintech): selfie video + spoken prompt
Account recovery / call-center verification: voice + face on video chat
Secure conferencing / impersonation defense: verify speaker identity in meetings
Step-up authentication for high-risk actions (payments, password reset)

4) Core research objectives
Objective A — Build baselines
Implement strong, publishable baselines for:
Face-only verification
Voice-only verification
Fusion (score fusion + learned fusion + cross-attention style fusion)
A strong fusion baseline to cite/replicate is the FG 2024 recursive joint cross-attention fusion approach.
Objective B — Deepfake stress testing (your main addition)
Systematically test these attacker scenarios:
Fake video + real audio
Real video + fake audio
Both fake
Cross-identity mismatch (face from ID A, voice from ID B)
Use a multimodal deepfake dataset like FakeAVCeleb and a speech spoof/deepfake benchmark like ASVspoof 2021.
Objective C — Find limitations and propose improvements
Identify and quantify:
Over-trust of corrupted modality
Poor calibration under attack (high confidence when wrong)
Failure under distribution shift (unseen generators, compression/noise)Then propose:
Uncertainty/quality-aware fusion

Optional: “fusion + deepfake detection gate” (video deepfake detector like MINTIME; AV deepfake detector like AVFF)

Threat Model and Test Matrix
7) Threat model
Assume the attacker wants to pass verification as a target identity.
Attack capabilities
A1: Video deepfake (face swap/re-enactment)
A2: Audio deepfake (voice cloning/conversion) or replay-style spoof
A3: Combined A/V deepfake
A4: Cross-identity mismatch (audio and face from different people)
The key is you are not just “detecting deepfakes” — you’re testing biometric verification failure under deepfakes.

How to Find Limitations (Research Method)
9) Limitation discovery checklist (what you measure)
9.1 Security failures
False Accept Rate (FAR) under each attack scenario
Attack Success Rate (ASR): % of attacked samples accepted
9.2 Usability failures
False Reject Rate (FRR) on genuine samples
Retry/step-up frequency (if you implement step-up)
9.3 Calibration failures (highly publishable)
Measure whether fused confidence is meaningful:
Expected Calibration Error (ECE)
Reliability curvesDeep insight: many systems stay “accurate” yet become overconfident under manipulations.
9.4 Generalization failures
Train fusion on one set of fake types, test on unseen fakes (FakeAVCeleb splits)
Add compression/noise sweeps:
Video: bitrate/codec changes
Audio: SNR sweep, reverberation
